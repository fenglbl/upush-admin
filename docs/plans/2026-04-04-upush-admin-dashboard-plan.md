# upush-admin 第一版实施计划

日期：2026-04-04
状态：Draft
设计稿：`docs/plans/2026-04-03-upush-admin-dashboard-design.md`
项目：`E:\Desktop\upush\upush-admin`

---

## 1. 实施目标

根据已确认设计，完成 `upush-admin` 第一版首页能力：

- 推送结果优先的数据看板首页
- 默认展示近 7 天趋势，可切换 `7d / 30d / 24h`
- 展示 KPI 卡片、趋势图、最近推送结果、异常日志摘要
- 提供少量快捷操作入口
- 后端通过专用 dashboard 接口提供聚合数据

---

## 2. 实施策略

本次实施按以下顺序推进：

1. **先补后端 dashboard 接口**
2. **再搭 admin 前端骨架**
3. **再接首页各模块与图表**
4. **最后联调、空状态、错误状态与视觉收尾**

原则：

- 第一版优先跑通主流程
- 不先做复杂权限体系
- 不先做底层模型重构
- 使用现有数据结构快速聚合

---

## 3. 任务拆分

### Task 1：梳理 dashboard 数据来源

**目标**
明确 `upushServer` 当前哪些数据可用于首页聚合。

**内容**
- 确认推送结果记录的来源与字段
- 确认是否已有可复用的推送状态记录
- 确认 `/logs` 能否稳定拿到 error / warn 摘要
- 确认近 7 天 / 30 天 / 24h 聚合所需最小字段

**产出**
- 一份可实现的 dashboard 数据映射说明

**完成标准**
- 明确 summary / trend / recentPushes / recentErrors 各自的数据来源

---

### Task 2：实现 `/admin/dashboard` 接口

**目标**
在 `upushServer` 新增首页专用聚合接口。

**内容**
- 新增路由：`GET /admin/dashboard`
- 支持参数：`range=7d|30d|24h`
- 返回固定结构：
  - `summary`
  - `trend`
  - `recentPushes`
  - `recentErrors`
  - `meta`
- 对非法 `range` 做兜底
- 增加基础异常处理与日志

**产出**
- 可稳定返回首页数据的 dashboard 接口

**完成标准**
- `7d / 30d / 24h` 都能返回有效 JSON
- 字段结构稳定
- 空数据时也能正常返回空数组/零值

---

### Task 3：初始化 `upush-admin` 前端项目骨架

**目标**
把 `upush-admin` 从空目录变成可开发的前端项目。

**内容**
- 选定技术栈（建议与当前生态保持一致，优先轻量、开发快）
- 初始化项目
- 建立基础目录结构
- 预留页面与组件目录
- 配置基础样式体系

**产出**
- 可运行的 admin 前端项目骨架

**完成标准**
- 本地能启动
- 能打开首页空壳页面

---

### Task 4：搭建首页静态布局骨架

**目标**
先把 Dashboard 首页结构搭起来，不急着接真数据。

**内容**
- Header 区
- KPI 卡片区
- 趋势图区
- 最近推送结果区
- 最近异常日志区
- 快捷操作区

**产出**
- Dashboard 首页静态布局

**完成标准**
- 首页结构完整
- 无数据时也有合理占位
- 响应式布局基本可用

---

### Task 5：接入 dashboard 接口

**目标**
让首页真正消费后端数据。

**内容**
- 封装 dashboard API 请求
- 首屏加载 dashboard 数据
- 支持 range 切换后重新请求
- 正确映射 summary / trend / lists

**产出**
- 首页真实数据接通

**完成标准**
- 首页能展示真实 KPI
- 切换 `7d / 30d / 24h` 时数据正确变化

---

### Task 6：实现趋势图

**目标**
完成首页核心折线图。

**内容**
- 接图表库
- 渲染三条线：
  - 总推送数
  - 成功数
  - 失败数
- 支持 `7d / 30d / 24h` 切换
- 处理空数据与少数据场景

**产出**
- 可用的趋势图模块

**完成标准**
- 图表渲染稳定
- 切换时间维度不卡顿
- 图例、颜色、时间轴清晰

---

### Task 7：实现最近推送结果列表

**目标**
补齐首页主业务列表区。

**内容**
- 展示最近推送记录
- 显示标题、时间、状态、摘要
- 状态颜色区分成功 / 部分失败 / 失败
- 预留跳转详情/记录页能力

**产出**
- 最近推送结果列表模块

**完成标准**
- 数据展示稳定
- 状态语义清晰
- 空状态正常

---

### Task 8：实现异常日志摘要区

**目标**
补齐首页辅助信息区。

**内容**
- 展示最近 error / warn
- 显示时间、级别、摘要
- 提供“查看全部日志”入口

**产出**
- 异常日志摘要模块

**完成标准**
- 首页可快速发现异常
- 不喧宾夺主

---

### Task 9：实现快捷操作区

**目标**
补齐少量工作台能力。

**内容**
- 新建推送入口
- 查看推送记录入口
- 查看日志入口

**产出**
- 快捷操作卡片/按钮区

**完成标准**
- 首页可直接跳关键页面
- 数量控制在少量范围内

---

### Task 10：联调与状态收尾

**目标**
完成第一版可用性收尾。

**内容**
- 加载状态
- 错误状态
- 空状态
- API 失败兜底
- 时间切换交互体验
- 视觉层级调整

**产出**
- 可演示、可继续扩展的第一版首页

**完成标准**
- Dashboard 首页在正常/空数据/异常场景下都可用

## 5. 第一版验收标准

满足以下条件即可认为第一版首页完成：

- 可以打开 `upush-admin` 首页
- 能看到 KPI 卡片
- 能看到趋势图
- 趋势图支持切换 `7d / 30d / 24h`
- 能看到最近推送结果
- 能看到最近异常日志摘要
- 能从首页进入关键后续页面
- dashboard 接口结构稳定
