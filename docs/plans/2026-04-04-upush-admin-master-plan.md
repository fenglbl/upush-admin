## 1. 产品定位

`upush-admin` 定位为 **推送运营与平台治理后台**，不是仅展示数据的看板。

核心目标：
1. 发得出去（推送执行）
2. 看得清楚（数据看板与记录）
3. 排得了错（日志与失败明细）
4. 管得住平台（用户/设备/配置/内容）

---

## 2. 功能全景

### A. 推送业务主线（已知计划）
1. Dashboard
2. 推送（新建/执行）
3. 推送记录（批次列表/详情）
4. 日志管理

### B. 平台管理主线（补充）
5. 用户管理
6. 设备管理
7. 协议管理
8. 版本管理

10. 系统设置

## 4. 总体开发流程

### Phase 1：打通业务闭环（发-看-查）
- Dashboard + 推送 + 推送记录 + 日志联动
- 输出：可真实运营、可快速排障

### Phase 2：补平台治理能力
- 用户管理 + 设备管理 + 反馈处理
- 输出：运营管理与问题处理流程可落地

### Phase 3：补内容与配置能力
- 协议管理 + 版本管理 + 系统设置完善
- 输出：后台具备长期运维能力

### Phase 4：工程化与稳定性
- 公共布局抽象
- 权限与审计完善
- 性能优化与回归机制
